Challenges

  • Datasets
    • Seeking data at the global level with very limited resources and outreach is a major limitation.
    • It is hard to find readily available global country-level datasets outside of major macro social and economic indicators produced or analyzed by international organizations such as World Bank.
    • More time would have been required to harvest data, cleanse and organize.
    • Often, when interesting datasets were found, by the owners/source websites forbid the use/reproduction of their datasets.
  • Tableau
    • While Tableau is a fantastic tool, the public version shows some connectivity instability. Because there is no option to save locally to the device, there is always the risk that some work may be lost during the save process to their public internet server.
  • Use of Python/Pandas Dataframes with Small Dataset
    • Because most of the datasets were available in spreadsheet format, this time Python was not the most practical tool to use to merge relatively short lists (less than 170 lines). However, should we decide to make time series analysis for the indicators, that would have been definitively mandatory.
  • HTML (Bootstrap vs. Jupyter Notebooks)
    • Bootrap CSS allows for faster templating of the website and responsive design, however the automatic CSS created for Jupyter notebooks to properly display as a web-page (see 'Statistics' link), conflicts with the Bootstrap navigation bar making for sub-optimal function and display.

Next Steps

  1. Control for region and maybe region X independent variables. Perhaps control for sex as well.
  2. Do search of literature and theory to guide what we do, what data we bring into the analysis and how to specify models.
  3. ML algorithm designed to predicted ranked lists?
  4. Time series/historical analyses of prior years' data.
  5. Use machine learning to capture and analyze psychological aspects of happiness.
